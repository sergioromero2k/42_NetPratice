Mandatory submission
What to submit to your Git repository

10 configuration files required

One file per level, exported with the "Get my config" button. They must be placed at the root of your repository. Make sure your login is entered before exporting each level.

README.md required

Written in English. Must follow the exact structure required by the subject. First line in italics with your login. Without this the project will not be evaluated.

README.md structure

First line req In italics: This project has been created as part of the 42 curriculum by <your_login>.
Description req What the project is about, its goal and a brief overview.
Instructions req How to launch the interface (run.sh or python3 -m http.server 49242), how to export configs, and that 10 files must be placed at the repository root.
Resources req References on TCP/IP, subnet masks, default gateways, routers/switches, OSI layers. Also a description of how you used AI and for which tasks.
Extra sections optional Usage examples, technical decisions, list of concepts learned, etc. Not required but adds value.
Evaluation / defense
How the defense works
During the defense
3 random levels
  • 3 random levels are generated on the spot
  • Limited time to solve them
  • No external tools allowed
  • Only a basic calculator like bc is tolerated
  • You must be able to explain every decision
To prepare
Practice these points
  • Calculate subnets by hand quickly
  • Identify the host range of a network
  • Know when a static route is needed
  • Understand what each editable field does
  • Use the "Evaluation" tab to simulate the defense

Do not copy solutions without understanding them. During the defense you will be asked why you set each value. If you cannot explain it, you fail.

Key interface tips

Intranet login

You must enter your login before exporting. Configs are generated specifically for you. Without the login the files are not valid for evaluation.

Level buttons

Check again verifies your config. Get my config downloads the file. Next level only appears after completing the level. Export BEFORE moving on.

Error logs

At the bottom of the page you will see logs. They tell you exactly what is wrong: missing gateway, invalid IP, unreachable destination, etc.

Editable fields

You can only modify the unshaded fields. Shaded fields are fixed. Adjust IPs, masks and routes in the available fields until everything is OK.

If run.sh fails

Run manually: python3 -m http.server 49242 and navigate to http://localhost:49242 in your browser.

Evaluation tab

Generates a random configuration. Use it to simulate the defense and practice under time pressure before the real day.
